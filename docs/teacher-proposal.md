# הצעה למורה - After Class AI

## תקציר

After Class AI היא הצעה להקים סביבת תרגול משותפת ללימודי הטמעת מערכות AI בארגונים. המערכת תאפשר לתלמידים להמשיך לתרגל אחרי שיעור Zoom בתוך כיתה וירטואלית חיה, עם סביבת n8n משותפת, תרגילים מוכנים, מעקב התקדמות, ותמיכה בזמן אמת.

הרעיון המרכזי: במקום שכל תלמיד יידרש לרכוש בעצמו מנויים, להקים שרתים, לפתוח חשבונות, לחבר API keys ולהסתבך עם תשתיות, בית הספר או תלמיד מוביל יכולים לספק סביבת תרגול אחת משותפת לכל הכיתה.

## הבעיה כיום

בלימודי AI ואוטומציה יש פער משמעותי בין ההסבר בשיעור לבין היכולת לתרגל בפועל.

כדי לתרגל ברצינות, תלמידים צריכים לעיתים:

- חשבון n8n פעיל או התקנה מקומית
- מנוי או API key ל-OpenAI / GPT
- מנוי או API key ל-Claude
- חיבור לכלים כמו Google Sheets, Gmail, Telegram, CRM ועוד
- הבנה טכנית של webhooks, credentials, הרשאות, docker ושרתים
- מחשב שמסוגל להריץ סביבת פיתוח

בפועל, לא לכל תלמיד יש:

- תקציב למנויים בתקופת הלימודים
- כרטיס אשראי לחיבור API
- ידע טכני להקים n8n לבד
- ביטחון לגעת בהגדרות שרתים ו-credentials
- זמן לפתור תקלות תשתית במקום לתרגל את החומר

התוצאה היא שחלק מהתלמידים מבינים את השיעור תאורטית, אבל לא מצליחים לתרגל מספיק בפועל.

## ההזדמנות

אם תהיה לכיתה סביבת תרגול משותפת, אפשר להפוך את הלמידה להרבה יותר מעשית.

במקום שכל תלמיד יקים הכל לבד, תהיה מערכת אחת:

- שרת n8n מרכזי
- תרגילים מוכנים מראש
- סביבת sandbox לתלמידים
- Google Sheets לניהול שיעורים ותלמידים בשלב MVP
- לוח מורה שמראה מי התקדם ומי צריך עזרה
- תצוגה חיה לשיתוף בזמן Zoom

כך התלמידים לומדים את מה שבאמת חשוב: בניית workflows, חשיבה אוטומטית, שילוב AI בתהליכים עסקיים, פתרון תקלות, והבנת שימושים ארגוניים.

## הפתרון המוצע

After Class AI היא שכבת ממשק מעל n8n.

היא לא מחליפה את n8n, אלא הופכת אותו לסביבת כיתה.

### המורה

המורה יכול:

- לפתוח שיעור חדש
- לבחור תרגיל n8n
- לשתף קוד הצטרפות בזום
- לראות מי הצטרף
- לראות מי הריץ workflow
- לזהות תלמידים שצריכים עזרה
- לשתף Live Board נקי בזמן השיעור

### התלמיד

התלמיד יכול:

- להיכנס עם קוד שיעור
- לקבל עותק אישי של workflow
- לפתוח את ה-workflow ב-n8n
- להריץ בדיקות
- ללחוץ "אני צריך עזרה"
- לראות את הסטטוס שלו

### הכיתה

הכיתה מקבלת:

- מעבדת תרגול משותפת
- תרגילים אחידים
- למידה מתוך טעויות אמיתיות
- שיתוף ידע בין תלמידים
- המשך עבודה אחרי שיעור Zoom

## למה שרת משותף

הקמת שרת משותף פותרת כמה בעיות בבת אחת.

### במקום שכל תלמיד יקים n8n

רק שרת אחד מריץ n8n.

התלמידים מתחברים אליו דרך הדפדפן, מקבלים workflows אישיים, ומתרגלים בלי התקנה מקומית.

### במקום שכל תלמיד ישלם על מנויים

אפשר להשתמש במודל מרוכז:

- API key אחד של בית הספר
- מכסת שימוש חודשית
- מודלים זולים לתרגול
- workflows עם נתוני דמו
- הגבלת שימוש לפי שיעור

כך אפשר לאפשר תרגול גם לתלמידים שאין להם מנוי פרטי.

### במקום שכל תלמיד ינהל credentials

המורה או בית הספר יכולים להכין credentials מבוקרים:

- Google Sheets דמו
- OpenAI key מוגבל
- Claude key מוגבל, אם משתמשים
- Webhooks פנימיים
- חשבונות sandbox בלבד

התלמיד מתרגל את הלוגיקה בלי לחשוף סודות אמיתיים.

## שתי אפשרויות הפעלה

### אפשרות 1 - שרת אישי שלי לכיתה

בשלב ראשון אפשר להקים שרת על חשבון אישי שלי לצורך פיילוט.

יתרונות:

- מתחילים מהר
- לא תלויים בהחלטות תשתית של בית הספר
- מתאים להוכחת יכולת
- אפשר להראות למורה ולכיתה דמו עובד

חסרונות:

- האחריות עליי
- צריך להגדיר גבולות שימוש
- לא מתאים לטווח ארוך בלי הסדרה
- יש לשמור על אבטחה ועלויות

### אפשרות 2 - שרת של בית הספר

בהמשך עדיף שבית הספר יפעיל את השרת.

יתרונות:

- בעלות רשמית של בית הספר
- ניהול משתמשים מסודר
- תקציב API מסודר
- אבטחה ובקרה טובים יותר
- אפשר לפתוח לכמה מחזורים וכיתות

חסרונות:

- דורש אישור והקמה
- דורש מישהו שיתחזק
- דורש החלטה על תקציב שימוש במודלי AI

## נקודות שחשוב לנהל נכון

### עלויות

צריך להגדיר:

- מכסת שימוש חודשית
- אילו מודלים מותר להפעיל
- כמה קריאות API מותרות בכל שיעור
- מי משלם על השרת ועל ה-API

### אבטחה

צריך להקפיד:

- לא לחשוף API keys לתלמידים
- לא לתת גישה מלאה לשרת
- לעבוד עם credentials מוגבלים
- לא להשתמש בנתונים רגישים אמיתיים
- לנקות workflows בסוף מחזור לימודים

### הרשאות

צריך להפריד בין:

- מורה
- תלמיד
- צופה
- מנהל מערכת

ב-MVP אפשר להתחיל פשוט, אבל לחשוב מראש על ההפרדה.

## למה להתחיל עם Google Sheets

Google Sheets מתאים לשלב הראשון כי:

- קל למורה לראות את הנתונים
- קל לשתף
- לא צריך לבנות בסיס נתונים מלא
- אפשר לערוך ידנית אם יש תקלה
- Apps Script יכול לתת API פשוט

בהמשך, אם המערכת תגדל, אפשר לעבור ל-Postgres או בסיס נתונים מלא.

## משפט הצגה קצר

After Class AI הוא רעיון להקים מעבדת תרגול משותפת ל-AI ו-n8n עבור תלמידי הקורס. המערכת באה לפתור את הפער שבו לא לכל תלמיד יש מנויים, API keys או יכולת להקים שרתים לבד. במקום זאת, הכיתה מקבלת שרת תרגול מרכזי, workflows אישיים לכל תלמיד, ולוח מורה שמראה בזמן אמת את ההתקדמות והקשיים. כך אפשר להפוך את שיעורי הזום מתצוגה חד-פעמית לסביבת עבודה חיה, משותפת ונגישה לכולם.
