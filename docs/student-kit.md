# Student Kit - ערכת הרצה לחניך

## המטרה

לכל חניך תהיה תיקיית תרגול מקומית שאפשר להוריד למחשב ולהמשיך לעבוד איתה גם מחוץ למערכת הכיתתית.

זה חשוב כי לא כל תרגול חייב להיות תלוי בשרת הכיתה. אחרי השיעור, החניך יכול לקחת את התבניות, לייבא אותן ל-n8n שלו, לשנות, לשבור, לתקן ולשמור גרסאות אישיות.

## מבנה הערכה

```text
student-kit/
  run-local.bat
  run-with-ngrok.bat
  index.html
  README.md
  config/
  payloads/
  templates/
  tools/
  my-workflows/
  notes/
```

## תפעול לחניך

1. מורידים את תיקיית `student-kit`.
2. לוחצים על `run-local.bat`.
3. הדפדפן נפתח בכתובת:

```text
http://127.0.0.1:5180/
```

אם אין Python, הקובץ פותח את `index.html` ישירות. זה מצב fallback כדי שהחניך לא ייתקע.

4. מורידים תבניות מהשיעור ושומרים ב-`templates/`.
5. משתמשים ב-payloads מוכנים מתוך `payloads/`.
6. מייבאים את הקבצים ל-n8n.
7. שומרים גרסאות אישיות ב-`my-workflows/`.

## עיקרון מוצר

הערכה חייבת לעבוד לחניך בלי רקע טכני:

```text
לוחץ על קובץ BAT -> נפתח דפדפן -> יש תבניות -> יש payloads -> אפשר לתרגל
```

אם חסר רכיב כמו Python או ngrok, הערכה צריכה להסביר מה חסר ולהמשיך במצב הכי שימושי שאפשר.

## ngrok

ngrok הוא אופציונלי.

מוסיפים אותו רק אם צריך:

- webhook ציבורי מהמחשב של החניך
- שיתוף תוצאה זמנית עם המורה
- בדיקה מול שירות חיצוני שלא יכול להגיע ל-127.0.0.1

מבנה:

```text
student-kit/tools/ngrok.exe
```

הפעלה:

```text
run-with-ngrok.bat
```

ברירת המחדל היא לא להשתמש ב-ngrok, כדי לא להעמיס על חניכים שלא צריכים את זה.

## איך תבניות יגיעו מהמערכת

בשלב הראשון:

- המורה שולח קובץ JSON בזום / WhatsApp / GitHub
- החניך שומר אותו ידנית ב-`templates/`

בשלב הבא:

- בממשק יהיה כפתור "הורד תבנית"
- הכפתור יוריד קובץ JSON מתוך ספריית התרגילים
- החניך ישמור אותו בתוך `student-kit/templates`

בהמשך מתקדם:

- המערכת תדע לארוז ZIP אישי לכל חניך
- ה-ZIP יכלול את תבניות השיעור, הערות, payloads לדוגמה וקובץ BAT

## מה לא לשים בערכה

- API keys
- credentials אמיתיים
- סיסמאות
- נתוני לקוחות אמיתיים

הערכה מיועדת לתרגול, לא לאחסון סודות.
