# בדיקת קצה לקצה - מורה ואור

## המטרה

לוודא לפני השיעור שהזרימה עובדת:

- הממשק עולה
- סביבת העבודה עולה
- אור נכנס
- אור מוריד תרגיל
- אור מייבא תרגיל
- אור מריץ תרגיל
- מתקבלת תשובה
- המורה רואה פעילות בלוח

## שלב 1 - הפעלת הכיתה

במחשב המורה:

```text
classroom-run/START_CLASS_TOMORROW.bat
```

להשאיר את כל החלונות פתוחים.

## שלב 2 - פתיחת הממשק

המורה פותח:

```text
http://127.0.0.1:5174/after-class-ai/
```

אור פותח:

```text
http://10.0.0.1:5174/after-class-ai/
```

או קישור ציבורי אם עובדים דרך מנהרה.

## שלב 3 - אור מצטרף

אור עובר למסך תלמיד.

שם:

```text
אור
```

קוד שיעור:

```text
AI-203
```

לוחץ:

```text
הצטרפות
```

בודקים בגיליון בלשונית:

```text
students
```

## שלב 4 - אור מוריד תרגיל

אור לוחץ:

```text
הורד workflow
```

אמור לרדת קובץ:

```text
after-class-current-workflow.json
```

זה תרגיל דמו:

```text
Customer Service Chatbot
```

## שלב 5 - אור פותח סביבת עבודה

אור פותח:

```text
http://10.0.0.1:5678/
```

או את קישור המנהרה של:

```text
n8n
```

## שלב 6 - אור מייבא את התרגיל

בתוך:

```text
n8n
```

אור מייבא את הקובץ שהוריד.

## שלב 7 - אור מריץ בדיקת לקוח

בצומת:

```text
Support Chat Webhook
```

לוחצים:

```text
Listen for test event
```

לאחר מכן שולחים בקשה דרך ממשק הבדיקה של:

```text
n8n
```

או דרך כלי הבדיקה המובנה אם זמין.

נתון בדיקה:

```json
{
  "name": "Or",
  "email": "or@example.com",
  "message": "שלום, אני רוצה לדעת איך מקבלים החזר כספי על הזמנה"
}
```

## תשובה צפויה

```json
{
  "customer_name": "Or",
  "detected_intent": "refund",
  "reply": "אפשר לבקש החזר עד 14 יום ממועד הרכישה. נציג יחזור אליך עם אישור.",
  "demo_note": "תשובה מובנית לדמו בלבד"
}
```

## איפה אור משנה משהו

אור פותח את הצומת:

```text
Student Edit Area - Canned Replies
```

ומוסיף תשובה חדשה.

לדוגמה:

```text
אחריות
```

או:

```text
warranty
```

המטרה היא להבין איך בונים לוגיקה פשוטה של שירות לקוחות.

## שלב 8 - דיווח ללוח

אם עדיין אין דיווח אוטומטי מתוך התרגיל, אור חוזר לממשק ולוחץ:

```text
הרץ workflow
```

זה שלב גיבוי.

המטרה הבאה היא להוסיף לתרגיל צומת דיווח אוטומטי.

## הצלחה בבדיקה

הבדיקה נחשבת מוצלחת אם:

1. אור נכנס לממשק.
2. אור הוריד את התרגיל.
3. אור ייבא אותו.
4. אור הריץ.
5. התקבלה תשובת שירות לקוחות.
6. הפעילות הופיעה בלוח או בגיליון.
