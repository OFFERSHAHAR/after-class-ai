# After Class AI - פירוט ממשק וכיווני המשך

## הרעיון

After Class AI הוא מרחב תרגול חי לתלמידים אחרי שיעור Zoom. במקום שכל תלמיד יתרגל לבד, הכיתה ממשיכה לעבוד יחד על תרגילי n8n, עם לוח מורה, סביבת תלמיד, תבניות workflow ופיד הרצות חי שכל המשתתפים יכולים לראות.

המטרה היא ליצור תחושה של מעבדת AI ארגונית: כל תלמיד מקבל עותק אישי של workflow, מריץ ניסויים, והמורה רואה מי מתקדם, מי צריך עזרה ואילו תקלות חוזרות על עצמן.

## מה קיים בממשק הנוכחי

### לוח מורה

מסך השליטה המרכזי של המורה:

- מספר תלמידים פעילים
- מספר הרצות מוצלחות
- מספר תלמידים שדורשים עזרה
- התרגיל הנוכחי בכיתה
- פיד חי של פעולות והרצות
- כפתור להרצת דמו
- כפתור לפתיחת מפגש
- כפתור לשכפול תרגיל לכל הכיתה

### מעבדה חיה

פיד שמציג בזמן אמת אירועים בכיתה:

- תלמיד שהצטרף
- workflow שהורץ בהצלחה
- תלמיד שצריך עזרה
- שגיאת JSON או credentials
- שכפול תרגיל לכל הכיתה

זה החלק שנותן לממשק תחושה של "כיתה אחרי הכיתה" ולא רק מערכת ניהול.

### מסך תלמיד

מסך שבו תלמיד יכול:

- להזין שם
- להצטרף למעבדה
- לראות workflow אישי לדוגמה
- להריץ workflow
- לפתוח סביבת n8n

כרגע זה דמו, אבל המבנה מתאים לחיבור אמיתי ל-n8n.

### תבניות n8n

אזור שמציג תרגילים מוכנים:

- Webhook Echo
- Lead Bot
- Ticket Router

כל תבנית יכולה בעתיד להפוך ל-workflow אמיתי שנשמר כ-JSON ומיובא אוטומטית ל-n8n.

## מה חסר כרגע

### חיבור אמיתי ל-n8n

כרגע הממשק מדמה הרצות. השלב הבא הוא לחבר אותו לשרת n8n מקומי:

- יצירת workflow מתוך תבנית
- שכפול workflow לתלמיד
- הפעלת workflow
- קריאת סטטוס execution
- הצגת שגיאות אמיתיות מההרצה

### משתמשים והרשאות

צריך להגדיר מודל הרשאות:

- מורה
- תלמיד
- צופה
- עוזר הוראה

חשוב שתלמידים לא יקבלו גישה מלאה לכל n8n, אלא רק לסביבת sandbox או workflows שהוקצו להם.

### חדרי שיעור

צריך לאפשר פתיחת חדר לכל מפגש:

- שם שיעור
- תאריך
- תרגיל נוכחי
- רשימת משתתפים
- קישור הצטרפות
- ארכיון הרצות מאותו מפגש

### מצב תצוגה לשיתוף מסך

כדאי להוסיף מסך "Live Board" נקי להצגה בזום:

- מי הצטרף
- מי הריץ בהצלחה
- מי תקוע
- איזה workflow פעיל
- גרף התקדמות כיתתי

זה יהיה המסך שהמורה משתף אחרי השיעור.

### ספריית תרגילים

כדאי לבנות מאגר תרגילים מסודר לפי רמות:

- מתחילים: Webhook, JSON, HTTP Request
- ביניים: Sheets, Email, Telegram, Forms
- AI: OpenAI, סיווג טקסט, סיכום, חילוץ מידע
- ארגוני: CRM, תמיכה, ניהול לידים, תיעוד פגישות

### אזור עזרה

כדאי להוסיף:

- כפתור "אני צריך עזרה"
- סיבת התקלה
- צילום או payload אחרון
- סטטוס טיפול של המורה
- פתרונות נפוצים

### תיעוד התקדמות

אפשר להוסיף לכל תלמיד:

- תרגילים שהושלמו
- שגיאות שנפתרו
- workflows שנוצרו
- הערות מהמורה
- badge או ציון התקדמות

## תוספות שיכולות להיות חזקות במיוחד

### כפתור "שכפל אליי"

תלמיד רואה תרגיל ולוחץ:

"שכפל אליי"

המערכת יוצרת לו עותק אישי ב-n8n, עם שם ברור כמו:

`Lead Bot - Noa - Session 03`

### replay של הרצה

אחרי שתלמיד מריץ workflow, הכיתה יכולה לראות:

- הקלט שנכנס
- הצמתים שעברו
- איפה הייתה שגיאה
- מה הפלט הסופי

זה יהפוך תקלות לכלי למידה משותף.

### תרגילים עם נתוני דמו

במקום שכל תלמיד ימציא payload:

- ליד לדוגמה
- פנייה מתמיכה
- תמלול שיחה
- הזמנה מחנות
- שורת CRM

לחיצה אחת שולחת נתוני דמו ל-webhook של התלמיד.

### מצב "מורה משתלט"

המורה יכול לפתוח workflow של תלמיד, לצפות בלבד או לקחת שליטה זמנית כדי להראות תיקון.

### דירוג מוכנות ארגונית

כיוון שאתה מלמד הטמעת AI בארגונים, אפשר להוסיף לכל תרגיל שכבה עסקית:

- מה הבעיה הארגונית
- מי המשתמש בארגון
- מה הסיכון
- אילו הרשאות נדרשות
- איך מודדים הצלחה

זה יבדל את המערכת מקורס טכני רגיל.

## מבנה MVP מומלץ לשלב הבא

1. שרת n8n מקומי עם Docker Compose
2. אפליקציית ממשק שמתחברת ל-n8n API
3. יצירת חדר שיעור
4. יצירת תלמידים ידנית או בקישור הצטרפות
5. תבנית workflow אחת אמיתית
6. שכפול workflow לתלמיד
7. הצגת execution status בלוח המורה

## המסר להצגה

After Class AI היא כיתה וירטואלית לתרגול מעשי של אוטומציות AI אחרי השיעור. המערכת מחברת בין למידה קבוצתית לבין סביבת n8n אמיתית: כל תלמיד מקבל workflow אישי לניסוי, והמורה רואה בזמן אמת את ההתקדמות, ההרצות והתקלות של כל הכיתה.

החזון הוא להפוך שיעורי AI מארוע חד-פעמי בזום למעבדת תרגול מתמשכת, שבה תלמידים בונים, מריצים, טועים, מתקנים ולומדים יחד.
