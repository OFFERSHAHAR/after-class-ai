# תוכנית הרצה עד מחר בבוקר

## המטרה

להגיע לשיעור מחר עם גרסה שעובדת בפועל לכיתה מלאה.

לא בונים עדיין את כל המערכת.

בונים גרסת שימוש ראשונה:

- תלמיד מצטרף עם שם וקוד שיעור
- ההצטרפות נרשמת בגיליון
- תלמיד מסמן הרצה
- תלמיד שולח בקשת עזרה
- המורה רואה מספר תלמידים, הרצות ובקשות עזרה

## מה חייב לעבוד מחר

### חובה

1. הגיליון מוקם ומעוצב.
2. הסקריפט מפורסם ככתובת רשת.
3. כתובת הסקריפט נמצאת בקובץ ההגדרות.
4. תלמיד יכול להצטרף.
5. ההצטרפות מופיעה בגיליון.
6. כפתור בקשת עזרה כותב שורה בגיליון.
7. לוח המורה מתעדכן.

### לא חובה למחר

- שכפול אמיתי של תהליכים.
- חשבונות משתמש מלאים.
- חיבור מלא לשרת אוטומציה.
- הרשאות מתקדמות.
- בדיקת עומס מקצועית.

## שלב 1 - הקמת הגיליון

פותחים את הגיליון:

```text
https://docs.google.com/spreadsheets/d/19GOHx2oq1ZsnkggJ-r-8TGvYvhYle3sAWifAzmL-rwQ/edit
```

נכנסים אל:

```text
Extensions > Apps Script
```

מדביקים את הקוד מתוך:

```text
google-apps-script/Code.gs
```

מריצים:

```javascript
setupAfterClassSheets()
```

מאשרים הרשאות.

## שלב 2 - פרסום הסקריפט

בעורך הסקריפטים:

1. לוחצים על פריסה.
2. בוחרים פריסה חדשה.
3. בוחרים אפליקציית רשת.
4. מריצים כמשתמש שלך.
5. נותנים גישה למי שיש לו קישור.
6. מעתיקים את כתובת הפריסה.

צריך לקבל כתובת שמתחילה בערך כך:

```text
https://script.google.com/macros/s/...
```

## שלב 3 - הכנסת הכתובת לממשק

פותחים את הקובץ:

```text
config.js
```

מכניסים את הכתובת כאן:

```javascript
sheetsApiUrl: "PASTE_URL_HERE",
```

שומרים.

מרעננים את הדפדפן.

אם הכל תקין, ההודעה במסך התלמיד תשתנה למצב מחובר.

## שלב 4 - בדיקת תלמיד אחד

במסך תלמיד:

1. מזינים שם.
2. מזינים קוד שיעור:

```text
AI-203
```

3. לוחצים הצטרפות.
4. בודקים שנוספה שורה בלשונית:

```text
students
```

5. לוחצים בקשת עזרה.
6. בודקים שנוספה שורה בלשונית:

```text
help_requests
```

7. לוחצים הרצה.
8. בודקים שנוספה שורה בלשונית:

```text
runs
```

## שלב 5 - בדיקת כיתה קטנה

לפני מחר, מבקשים משניים או שלושה חברים להיכנס.

כל אחד:

1. נכנס לקישור.
2. כותב שם.
3. כותב קוד שיעור.
4. לוחץ הצטרפות.
5. לוחץ בקשת עזרה.
6. לוחץ הרצה.

בודקים שהלוח מתעדכן.

## שלב 6 - הכנה לשיעור מחר

לפני השיעור:

1. פותחים את לוח המורה.
2. פותחים את הגיליון בחלון נוסף.
3. פותחים את המצגת למורה אם צריך להציג את הרעיון.
4. מכינים הודעה לתלמידים.

הודעה מוכנה:

```text
נכנסים לקישור, עוברים למסך תלמיד, כותבים שם וקוד שיעור AI-203, ואז לוחצים הצטרפות.
אחרי ההצטרפות לחצו פעם אחת על הרצה ופעם אחת על בקשת עזרה כדי שנבדוק שהמערכת עובדת.
```

## שלב 7 - בזמן השיעור

מה עושים:

1. משתפים את לוח המורה.
2. שולחים לתלמידים את הקישור.
3. מבקשים להצטרף.
4. בודקים שמספר התלמידים עולה.
5. מבקשים ללחוץ הרצה.
6. מבקשים ממי שצריך ללחוץ בקשת עזרה.
7. מסבירים שזה שלב ראשון חינמי של מעבדה כיתתית.

## תוכנית גיבוי

אם החיבור לגיליון לא עובד:

1. משתמשים במצב דמו.
2. פותחים את הגיליון ידנית.
3. מבקשים מתלמידים להירשם בטופס ידני או בצאט.
4. עדיין מציגים את הרעיון ואת ערכת החניך.

אם הממשק לא עולה:

1. פותחים את המצגת.
2. מציגים את התוכנית.
3. שולחים את ערכת החניך.

## הדבר הכי חשוב

מחר לא צריך להוכיח שהמערכת מושלמת.

צריך להוכיח שהכיתה יכולה לעבוד סביב רעיון אחד:

סביבת תרגול משותפת, חינמית להתחלה, שמורידה חסמים מהתלמידים ונותנת למורה תמונת מצב.
